# In-Memory База Данных

Локальное хранилище данных формата «ключ-значение», написанное на C++. Проект работает в оперативной памяти и предназначен для быстрого кэширования информации. 

## Описание

- Поиск, добавление и удаление элементов происходят за время O(1). Это достигается за счет связки хэш-таблицы и двусвязного списка.
- Ограничение памяти (LRU): У базы есть жесткий лимит записей. Если места не хватает, программа автоматически удаляет самый старый элемент.
- Удаление по времени (TTL): Каждой записи можно задать «срок годности». База проверяет время жизни элемента в момент обращения к нему и удаляет данные, если они просрочены.

## Структура папок

- `include/` - хранится заголовочный файл.
- `src/` - хранятся сам код базы и файл для ручных проверок (`in_memory_db.cpp`, `main.cpp`).
- `tests/` - хранится файл с тестами от Google.

## Как собрать проект

Для запуска нужно ввести эти команды:

```bash
mkdir build
cd build
cmake ..
cmake --build .
./db_cli
```

Команды:
- put (ключ) (значение) - через пробел - добавить бессрочную запись
- put (ключ) (значение) (х) - добавить запись, которая исчезнет через х миллисекунд.
- get (ключ) - получить значение по ключу. Если времени прошло слишком много или ключа нет, вернется null.
- exit - выйти из программы.

## Запуск автоматических тестов

Проект покрыт тестами на базе фреймворка Google Test, которые проверяют логику удаления и лимиты памяти. 
Чтобы их запустить:

```bash
cd build
ctest
```
